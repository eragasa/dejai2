# note-lifecycle-and-containment.00

**Purpose.**  
This note defines a **hard control system** for managing idea proliferation in Obsidian-based research notebooks. It treats notes as stateful objects with explicit transitions, preventing exploratory thinking from polluting canonical knowledge. This is a structural, not motivational, intervention.

This policy is binding for the `dejai2` knowledge base.

---

## Core Diagnosis

Unbounded growth occurs when:
1. ideas are stored at the wrong resolution,
2. foundational notes do not act as load-bearing interfaces,
3. exploratory material leaks into canonical space,
4. notes have no lifecycle or promotion rules.

The remedy is **explicit state, directionality, and containment**.

---

## Note State Model (Mandatory)

Every note MUST belong to exactly one state.

```text
scratch → exploratory → structured → canonical → frozen
